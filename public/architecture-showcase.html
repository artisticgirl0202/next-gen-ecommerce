Data Flow frontend / backend / db 
Frontend (React: ProductList)
  └─ useEffect → fetchProducts(1, 600)
       └─ fetchProductsPage → GET /api/products?page=1&page_size=600
Backend (FastAPI: /api/products)
  ├─ Redis items:list cache lookup
  ├─ Cache hit → return items
  └─ Cache miss → query PostgreSQL products
       └─ On success → re-cache in Redis (1h) → return items
Response (items/products)
  └─ fetchProducts: normalize products/items → mapDtoToProduct
  └─ updateProductIndexes → merge into ALL_PRODUCTS
  └─ setProductsKey / setProductsCount → render UI

The frontend calls fetchProducts to request the list. The backend checks Redis first, then falls back to PostgreSQL on a miss. The response is normalized via mapDtoToProduct, merged into ALL_PRODUCTS, and then re-rendered.

Challenge 01: Latency

Response Time Bottleneck

Problem: Running model training and logging synchronously on order requests causes API response time to spike.

Solution Async Event-Driven Architecture
backend/services/order_service.py
# 1. DB Save (Blocking, Fast)
db.add(order); db.commit()

# 2. Publish Event (Non-blocking)
try:
  send_user_event(payload, topic="order.events")
except Exception:
  logger.exception("Fallback to file")

→ Fire-and-forget events to Kafka keep API responses under 200ms.

Challenge 02: Accuracy

Static Vector Limit

Problem (code-verified): FAISS similarity alone cannot capture real-time user interest from recent clicks.

Solution Hybrid Re-ranking Engine
backend/routers/recommend.py
# 1) Static Similarity (FAISS)
idxs, scores = FAISS.search(query_vec, top_k=cand_k)

# 2) Dynamic Boosts (Redis click log)
boosts_list = boost_by_recent_clicks(u_id, candidate_pids)

# 3) Hybrid Re-ranking
final_scores = arr_norm[:min_len] + boosts[:min_len]

→ The production code combines FAISS similarity with Redis-based click boosts to raise real-time personalization.

Challenge 03: Reliability

System Failure Handling

Problem: If the ML server or Redis fails, the storefront returns 500 errors.

Solution Graceful Degradation
backend/routers/recommend.py
try:
  return recommend_hybrid(...)
except Exception:
  logger.error("ML System Failure")
  return get_fallback_from_db(db)

→ On failures, fall back to popular products from DB to preserve availability.
